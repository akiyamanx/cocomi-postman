## 🔥 NEW! COCOMI自動修復CI構想（2026-02-28 朝のひらめき）

### 概要
CI（COCOMI CIテスト審査）が不合格になった時、人間の介入なしにClaude Codeが自動で修正して再pushし、合格するまで繰り返すシステム。

### 発想のきっかけ
メンテナンスマップv2.4のデプロイで、data-storage.jsが500行超過→CIテスト失敗→GitHub Pagesにデプロイされない問題が発生。原因特定に時間がかかった（Postmanの無料枠切れでLINE通知が来なかった）。もしCIの結果がClaude Codeに自動で戻って修正→再pushしてくれたら、人間は待ってるだけでよかった。

### 理想フロー
```
LINE指示（例：「このファイル修正して」）
  ↓
Cloudflare Workers（COCOMI Postman v1.4）
  ↓
GitHub（ファイル更新 or dispatch）
  ↓
Claude Code（タブレットTermux）→ 修正 → git push
  ↓
COCOMI CI テスト審査（GitHub Actions）
  ├── ✅合格 → GitHub Pages自動デプロイ → LINE通知「✅合格！デプロイ完了」
  └── ❌不合格 → CIログ取得
                    ↓
              Cloudflare Workers が CIエラー内容を受信
                    ↓
              Claude Code に「ここで不合格。修正して再push」と自動指示
                    ↓
              Claude Code が修正 → 再push → CIが再度走る
                    ↓
              （合格するまでループ。最大N回でタイムアウト）
                    ↓
              LINE通知（毎回の結果 + 最終結果）
```

### 技術要素（必要なもの）
1. **GitHub Actions → Cloudflare Workers へのwebhook**
   - CI失敗時にワークフローからCloudflare Workersのエンドポイントを叩く
   - エラー内容（500行超過、console.log残り等）をJSON本文で送信
2. **Cloudflare Workers の拡張（Postman v1.5?）**
   - CI結果を受信するエンドポイント追加
   - 「修正指示」をGitHub repository_dispatchで再度Claude Codeに送信
   - ループ回数の管理（無限ループ防止）
3. **Claude Code側の対応**
   - CI失敗内容を読み取って該当箇所を自動修正
   - 修正→commit→push のフローを自動実行
4. **LINE通知の拡張**
   - 不合格時もCI結果の詳細をLINEに通知
   - リトライ中の状況もLINEに送信
   - 最終結果（合格 or タイムアウト）をLINEに送信

### 実装の優先度
- **依存:** COCOMI Postman v1.4（現在完成済み）、cocomi-ci.yml（全リポジトリ導入済み）
- **前提:** Postmanの無料枠回復（LINE通知が必要）
- **難易度:** 中〜高（webhook連携＋ループ制御＋エラーパース）
- **インパクト:** 超高！完全自動開発サイクルの実現。KatariCodeの究極進化形

### 類似サービスとの比較
- **GitHub Copilot coding agent**: 似た機能（CI失敗→自動修正→再PR）があるが有料、PC向け
- **COCOMI自動修復CI**: スマホ＋Termux環境で動作、LINE連携、アキヤの業務フローに特化
- **アドバンテージ**: COCOMI CIのルール（500行制限等）を熟知したClaude Codeが修正するため、精度が高い

### ステップ分割案
| Step | 内容 | 依存 |
|------|------|------|
| Step1 | CI失敗時にLINEへエラー詳細を通知（現状は成功/失敗のみ） | cocomi-ci.yml修正 |
| Step2 | CI失敗時にCloudflare Workersへwebhook送信 | Step1 + Postman拡張 |
| Step3 | Cloudflare WorkersからClaude Codeへ修正指示の自動送信 | Step2 |
| Step4 | Claude Codeでの自動修正→再push | Step3 |
| Step5 | ループ制御（最大リトライ回数、タイムアウト） | Step4 |
| Step6 | LINE通知の拡張（リトライ状況、最終結果） | Step5 |
