#### 技術メモ
- **PDF方式**: jsPDF廃止 → `window.open()` + `window.print()` でHTML→PDF
  - メリット: フォントファイル不要、CSSで書式自由、軽い
  - 制約: 印刷ダイアログ経由（「PDFとして保存」選択が必要）
- **印刷環境**: RICOH P C301SF (192.168.11.100) で確認済み
- **書式**: 見積書・請求書は同じHTMLテンプレートでタイトル変えるだけ
- **アキヤの好み**: A4縦、行数多め（25〜30行）、材料多め対応

## ▼ 5. 思想・ルール・方針の追加

```text
- **★【NEW!】Canvas vs DOM の使い分けルール:**
  - 手書きの線（フリーハンド描画）→ Canvas（高速描画、後から個別操作不要）
  - テキスト・ピンなど後から操作したい要素 → DOM要素（移動・サイズ変更・編集が可能）
  - エクスポート時はDOMもCanvasに描き直して合成する

- **★【NEW!】座標変換のベストプラクティス:**
  - getBoundingClientRect()で図面要素を直接参照
  - コンテナ基準ではなく図面要素基準で計算
  - scale/translateの個別逆算は不要（rectに反映済み）
```

### Claude Code /tmp問題（重要な背景知識）
- Claude Codeは内部で/tmp/claude/をハードコードして使用
- Termux(Android)では/tmpにアクセス権限がない
- git pushなど長時間処理が失敗する
- ポストマンの解決策: executor.sh v1.1で「git pushはTermuxから直接実行」設計に変更
- これにより世界中のTermuxユーザーが困っている問題をポストマンが自動解決
- パッケージ販売時のアピールポイント

## ▼ 4. Architecture Notes への追記

### フォント関連
```text
- NotoSansJPサブセット: fontTools(pyftsubset)でレシート頻出約1,140文字に絞ったサブセットTTFを生成→Base64変換→noto-sans-jp-base64.jsに格納
- 含有文字: ひらがな・カタカナ・ASCII(0x20-0x7E)・全角英数(0xFF10-0xFF5A)・半角カタカナ・レシート頻出漢字約800字・記号・¥(U+00A5)・￥(U+FFE5)
- サイズ: TTF 304KB → Base64 405KB（PWAキャッシュ許容範囲）
- フォント登録手順: doc.addFileToVFS('NotoSansJP-Regular.ttf', NOTO_SANS_JP_BASE64) → doc.addFont(...) → doc.setFont('NotoSansJP')
- フォールバック: NOTO_SANS_JP_BASE64未定義時はhelveticaで代替（日本語は文字化けするが動作は壊れない）
```

### 確立した設計パターン追記
```text
- **フォントサブセット化パターン:** fontTools(pyftsubset)でUnicodeコードポイント指定→サブセットTTF→Base64→JSファイル格納。通貨記号(¥,$等)は明示的に含める
- **PDF上デバッグパターン:** スマホ環境でconsole.logが使えない時、PDF上にdoc.setFontSize(7)+setTextColor(180,0,0)で赤字デバッグ情報を描画→原因特定後にコード削除
- **ページ跨ぎ防止パターン:** 複数要素(テキスト+画像)を1ブロックとして扱い、描画前にブロック全体の高さを概算→ページ残量と比較→収まらなければ先に改ページ
- **画像重複防止パターン:** attachedImagesオブジェクトでデータURL先頭100文字をキーに追跡→同一画像の2回目以降の添付をスキップ
- **Blobダウンロードパターン:** Base64→atob→Uint8Array→Blob→URL.createObjectURL→a.click()でダウンロード。スマホChromeでPDFを直接表示するより確実
```

## ▼ 4. Architecture Notes への追記

### フォント関連
```text
- NotoSansJPサブセット: fontTools(pyftsubset)でレシート頻出約1,140文字に絞ったサブセットTTFを生成→Base64変換→noto-sans-jp-base64.jsに格納
- 含有文字: ひらがな・カタカナ・ASCII(0x20-0x7E)・全角英数(0xFF10-0xFF5A)・半角カタカナ・レシート頻出漢字約800字・記号・¥(U+00A5)・￥(U+FFE5)
- サイズ: TTF 304KB → Base64 405KB（PWAキャッシュ許容範囲）
- フォント登録手順: doc.addFileToVFS('NotoSansJP-Regular.ttf', NOTO_SANS_JP_BASE64) → doc.addFont(...) → doc.setFont('NotoSansJP')
- フォールバック: NOTO_SANS_JP_BASE64未定義時はhelveticaで代替（日本語は文字化けするが動作は壊れない）
```

### 確立した設計パターン追記
```text
- **フォントサブセット化パターン:** fontTools(pyftsubset)でUnicodeコードポイント指定→サブセットTTF→Base64→JSファイル格納。通貨記号(¥,$等)は明示的に含める
- **PDF上デバッグパターン:** スマホ環境でconsole.logが使えない時、PDF上にdoc.setFontSize(7)+setTextColor(180,0,0)で赤字デバッグ情報を描画→原因特定後にコード削除
- **ページ跨ぎ防止パターン:** 複数要素(テキスト+画像)を1ブロックとして扱い、描画前にブロック全体の高さを概算→ページ残量と比較→収まらなければ先に改ページ
- **画像重複防止パターン:** attachedImagesオブジェクトでデータURL先頭100文字をキーに追跡→同一画像の2回目以降の添付をスキップ
- **Blobダウンロードパターン:** Base64→atob→Uint8Array→Blob→URL.createObjectURL→a.click()でダウンロード。スマホChromeでPDFを直接表示するより確実
```
