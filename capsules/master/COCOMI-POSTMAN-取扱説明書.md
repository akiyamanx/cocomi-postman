## 📱 LINEリッチメニュー（v2.1〜）

画面下部の **「📮 COCOMIメニュー」** をタップするとリッチメニューが開閉する。
6つのボタンで主要機能にワンタップでアクセスできる。

| ボタン | 送信されるコマンド | 機能 |
|---|---|---|
| ポストマン | 状態 | Workerのバージョン・機能一覧を表示 |
| カプセル | カプセル | カプセル保管庫を日付順Flexで表示 |
| アイデア | アイデア一覧 | アイデア保管庫の4カテゴリを表示 |
| フォルダ | フォルダ一覧 | 全フォルダ構造と件数を表示 |
| 指示を送る | — | テキスト入力画面が開く（指示入力用） |
| ヘルプ | ヘルプ | コマンド一覧を表示 |

## 📲 新しい操作方法（v2.2〜v2.4）

### カプセルの閲覧（Flex Messageタップ式）
1. **「カプセル」** をタップ（またはテキスト送信）
2. ファイル一覧がボタン式で表示される（日付ごとに緑色の見出し 📅 MM-DD）
3. **ファイル名ボタンをタップ** → ファイルの内容が表示される
4. **「もっと見る」ボタン**（緑色）で次の10件を表示

### フォルダの閲覧
1. **「フォルダ」** をタップ → 全フォルダ構造が表示
2. LINEで **「フォルダ capsules/daily」** のように送信 → そのフォルダの中身がFlex表示
3. ファイル名ボタンをタップ → ファイルの内容が表示される

### ファイル内容の直接表示
**「読む ○○」** でGitHub上のファイル内容をLINEで直接読める（5000字制限）。
例: 「読む capsules/daily/2026-02-26_DIFF総合.md」

### ヘルプの使い方
- **「ヘルプ」** → コマンド一覧を表示
- **「ヘルプ 指示」** → 指示書の書き方ガイドを表示

### 🔨 アプリを修正・機能追加したい

**簡単な修正（1ステップで終わるもの）：**

LINEで以下のように送る：
```
プロジェクト名: やりたいこと
```

例：
```
genba-pro: ログイン画面の背景色を青に変えて
culo-chan: 材料名の候補が表示されないバグを直して
maintenance-map: 地図の初期表示位置を東京にして
```

プロジェクト名を省略すると genba-pro に送られる。

**複雑な修正（マルチステップ）：**

① クロちゃん（claude.ai）に相談して指示書を作ってもらう
② クロちゃんが作ったファイルをダウンロード
③ LINEでそのファイルを送信
④ あとは全自動（ステップ実行→CI→完了通知）

### 📂 GitHubのフォルダ・ファイルを確認したい

| LINEで送る言葉 | 見れるもの | 返信形式 |
|---|---|---|
| **カプセル** | カプセル保管庫（日付順） | Flex（タップ式） |
| **カプセル 2** | カプセル2ページ目 | Flex（タップ式） |
| **フォルダ一覧** | 全体のフォルダ構造と件数 | テキスト |
| **フォルダ capsules/daily** | dailyフォルダのファイル一覧 | Flex（タップ式） |
| **フォルダ missions** | missionsのサブフォルダ一覧 | Flex（タップ式） |
| **アイデア一覧** | アイデア保管庫の状態 | テキスト |
| **読む ○○** | ファイルの内容（5000字制限） | テキスト |
| **状態** | Workerのバージョンと機能一覧 | テキスト |
| **ヘルプ** | コマンド一覧 | テキスト |
| **ヘルプ 指示** | 指示書の書き方ガイド | テキスト |

**重要:** 「フォルダ ○○」は「フォルダ」＋**半角スペース**＋フォルダ名。
スペースなしだとテキスト指示扱いになるので注意。

---

### 📋 指示書を正しい場所に届けたい

クロちゃんが作る指示書の先頭に以下を入れる：
```html
<!-- dest: missions/cocomi-postman -->
```

このdestタグがあると、Workerがファイルの中身を読んで指定先に自動配置する。
新しいフォルダ名を指定しても、自動でフォルダが作られる。

destタグがないファイルは：
- ファイル名キーワードにマッチ → 該当フォルダ
- マッチしない → inbox/ に保管

## 🔧 タブレットの操作

### 自動モードの起動
```bash
cd ~/cocomi-postman
./postman.sh auto
```

### 自動モードの状態確認
ミッションが検知・実行されるとLINEに通知が来る。

### /tmp権限エラーが出たら
```bash
export TMPDIR=~/tmp && mkdir -p ~/tmp
```

### Claude Codeがgit pushできない時
```
/exit
```
でClaude Codeを終了 → 手動で `git push`

## ☁️ Workerの更新手順

1. クロちゃんが新しいworker.jsを作成
2. ファイルをスマホにダウンロード
3. Chromeで Cloudflare Dashboard を開く
   - Workers & Pages → cocomi-worker → Edit code
4. 全選択 → 削除 → 新しいコードを貼り付け
5. 「Save and Deploy」をクリック
6. LINEで「状態」と送って、バージョン確認

## ⚠️ トラブルシューティング

### LINEで送ったのに返信が来ない
- Cloudflare Workerがダウンしている可能性
- Cloudflare Dashboard でWorkerのログを確認
- 「状態」と送って反応を確認

### 指示書を送ったのにタブレットで実行されない
- タブレットでpostman.sh autoが動いているか確認
- `git pull` を手動で実行して指示書が来ているか確認
- config.jsonのプロジェクト設定が正しいか確認

### CIが失敗した
- LINE通知のGitHub Actionsリンクをタップ
- どのテストで失敗したか確認
- Claude Codeがリトライするか、手動で修正

### 「フォルダ○○」が指示扱いになった
- 「フォルダ」と「○○」の間に**半角スペース**を入れる
- 例: 「フォルダ capsules/daily」（○）
- 例: 「フォルダcapsules/daily」（×→指示扱い）

## 📌 できること・できないことまとめ

### ✅ できること
- LINEテキストで簡単な修正指示を送る
- LINEファイルで複雑な指示書を送る
- カプセル（思い出・開発・マスター・企画書）を自動保管
- アイデアをカテゴリ別に保管
- GitHubのフォルダ構造をLINEで確認
- フォルダの中身（ファイル一覧）をLINEでFlex表示（タップ式）
- LINEからGitHub上のファイルの中身を読む（「読む ○○」コマンド、v2.1〜）
- リッチメニューからワンタップで主要機能にアクセス（v2.1〜）
- カプセル一覧をページネーションで閲覧（v2.3〜）
- 日付ごとに整理された見やすいファイル一覧（v2.4〜）
- ヘルプコマンドでコマンド一覧・指示の書き方を確認（v2.1〜）
- destタグで新しいフォルダを自動作成して配置
- マルチステップ指示書の全自動実行
- CI/CD自動テスト＆結果のLINE通知

### ❌ できないこと（現時点）
- 画像ファイルの配達（.mdテキストのみ対応）
- LINEからファイルを削除・移動
- 複数リポジトリへの同時配達
- タブレットの状態をLINEから確認する
