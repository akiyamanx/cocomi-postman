## 技術背景（次のクロちゃんが知るべきこと）

### SAM2のアーキテクチャ
```
エンコーダー（134MB、ORT形式）
  入力: [1, 3, 1024, 1024] float32（RGB正規化: (px/255)*2-1）
  出力: image_embed, high_res_feats_0 [1,32,256,256], high_res_feats_1 [1,64,128,128]
  推論: 21〜28秒（Galaxy S22, WASM）

デコーダー（20.6MB、ONNX形式）
  入力: image_embed, high_res_feats_0/1, point_coords, point_labels, mask_input, has_mask_input
  ★★★ orig_im_sizeは不要！（SAM1との違い）★★★
  出力: masks [1, 3, 256, 256], iou_predictions [1, 3]
  推論: 213〜334ms
```

### マスク座標変換（超重要！）
```
マスクは256×256で出力される（1024×1024ではない！）

タップ座標の変換:
  元画像座標 × scale + offset = 1024座標 → デコーダーに渡す

マスクの変換:
  マスク座標(256) × (1024/256) = 1024座標
  1024座標 - offset = スケール画像座標
  スケール画像座標 / scale = 元画像座標

オーバーレイ描画:
  maskScale = 256 / 1024 = 0.25
  srcX = offsetX × maskScale
  srcY = offsetY × maskScale
  srcW = imgWidth × scale × maskScale
  srcH = imgHeight × scale × maskScale
```

### ONNX Runtime Web
```html
<script src="https://cdn.jsdelivr.net/npm/onnxruntime-web@1.20.1/dist/ort.min.js"></script>
```
- executionProviders: `['wasm']`（最も互換性が高い）

### 画像前処理
```javascript
// アスペクト比維持、1024×1024にリサイズ、黒パディング
const scale = Math.min(1024 / img.width, 1024 / img.height);
const offsetX = (1024 - img.width * scale) / 2;
const offsetY = (1024 - img.height * scale) / 2;
// 正規化: (pixel / 255.0) * 2 - 1
```
