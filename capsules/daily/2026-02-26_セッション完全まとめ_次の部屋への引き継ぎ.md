## 🔑 今回確定した技術知見

### 1. Python + bash の役割分担
- bash: UI（メニュー表示、ユーザー入力、色付き出力）
- Python: データ操作（JSON読み書き、バックアップ、型変換、マスク）
- bashからpythonを `python3 script.py args` で呼び出す連携パターン

### 2. config.json安全編集パターン
- 書き込み前にconfig.json.bakへ自動バックアップ
- 書き込み失敗時はバックアップから自動復元
- デフォルトプロジェクトは削除不可（安全装置）
- _updatedフィールドに更新日付を自動記録

### 3. ログ統合表示パターン
- logs/execution/ + reports/ + errors/ + inbox/ を統合
- ファイルのタイムスタンプ（date -r）で時系列ソート
- 「タイムスタンプ|タイプ|プロジェクト|名前|パス」のパイプ区切り形式で内部処理

### 4. echo -e の注意点（Termux）
- `echo -n` は色エスケープコードを解釈しない
- `echo -e` を使うか、表示を別行に分ける必要がある
- サブシェル内の配列操作は親シェルに反映されない→先に配列構築

### 5. Claude Code /tmp問題（Termux特有）
- Claude Codeが/tmpにハードコードでアクセス→Termuxでは権限なし
- git pushなど長い処理で失敗する
- ポストマンの解決策: executor.shでTermuxから直接git push
- GitHub Issues: #15628, #15637, #16955, #17366, #18342
