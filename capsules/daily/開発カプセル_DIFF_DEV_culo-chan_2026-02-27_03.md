# 3. 🧠 Design Decisions（設計判断の記録）

### 設計判断①：複数画像の取得方法 — multiImageDataUrls優先＋フォールバック
- **課題:** 複数選択時に画像がPDFに出ない
- **原因:** generateAndSaveReceiptPdfs()がreceiptImageData（単枚用）しか参照してなかった
- **選択肢:**
  - A: multiImageDataUrlsだけ参照 → 単枚撮影時に画像が取れない
  - B: multiImageDataUrls優先、なければreceiptImageDataにフォールバック
- **決定:** B
- **理由:** 単枚・複数の両方のフローに対応。既存の単枚撮影を壊さない

### 設計判断②：レシートと画像の紐付け方法
- **課題:** 5枚の画像と3つのレシートをどう紐付けるか
- **決定:** レシートi番目に画像配列のi番目を対応。画像数を超えたら最後の画像をフォールバック
- **コード:** `var imgIdx = Math.min(i, allImages.length - 1);`
- **理由:** 1対1対応が基本。複数レシートが1枚の写真から認識されるケース（今の実態）に対応

### 設計判断③：AI重複レシート防止 — プロンプト修正のみ
- **課題:** 駐車証明書＋領収書が2件に認識される
- **選択肢:**
  - A: プロンプトで制御（Geminiに「まとめろ」と指示）
  - B: レスポンス後処理で重複検出＆マージ
- **決定:** A（プロンプト修正のみ）
- **理由:** 最もシンプル。プロンプト1行追加だけで効果検証できる。ダメなら後処理を追加すればいい
- **結果:** 一発で解決。後処理は不要だった

### 設計判断④：Termuxコマンド方式 — ファイルダウンロード＋cp
- **課題:** Pythonワンライナーで`[]`がbashに解釈されてエラー
- **選択肢:**
  - A: エスケープを工夫してPythonワンライナーを使う
  - B: Pythonスクリプトファイルを作ってから実行
  - C: 修正済みファイルをダウンロード＋cpで差し替え
- **決定:** C
- **理由:** 最も確実。エスケープの問題が起きない。ファイル全体の整合性も保証される

# 5. 🐛 Errors & Solutions（エラー＆解決策）

### エラー①: 複数選択時に画像がPDFに出ない → multiImageDataUrls参照追加
- **症状:** 「まとめて選択」でPDF生成すると画像なし
- **確定原因:** generateAndSaveReceiptPdfs()がreceiptImageData（単枚用）のみ参照。multiImageDataUrls（複数選択用）を見ていない
- **解決:** multiImageDataUrls優先＋receiptImageDataフォールバックのロジックに修正
- **パターンID:** ERR-CULO-0008
- **学び:** グローバル変数が複数ある場合、全ての入力パスで正しい変数が参照されているか確認

### エラー②: 駐車証明書＋領収書が2件に認識される → Geminiプロンプト調整
- **症状:** 同じ駐車場のレシートが2/2で表示
- **確定原因:** Geminiが駐車証明書と領収書を別レシートとして認識
- **解決:** プロンプトに「同一取引なので1つにまとめる」ルール追加
- **パターンID:** ERR-CULO-0009
- **学び:** AIの認識パターンはプロンプトで制御可能。まずプロンプト修正を試し、ダメなら後処理を追加

### エラー③: Termuxでbash履歴展開エラー → ファイルダウンロード方式に変更
- **症状:** `bash: !dateGroups[dateKey]: event not found`
- **確定原因:** Pythonワンライナー内の`[]`をbashが履歴展開として解釈
- **解決:** ファイルダウンロード＋cp方式に変更
- **パターンID:** ERR-TERM-0001
- **学び:** Termuxでの複雑なインラインコードは`[]`や`!`の解釈問題がある。ファイル差し替え方式が確実
