## 🔑 今回確定した技術知見

### 1. 複数選択時の画像データ受け渡し
- **原因:** receipt-pdf.jsがreceiptImageData（単枚用）のみ参照
- **解決:** multiImageDataUrls優先＋receiptImageDataフォールバック
- **コード:** `var imgIdx = Math.min(i, allImages.length - 1);`

### 2. Geminiプロンプトによるレシート重複防止
- **原因:** 駐車証明書と領収書が別レシートとして認識
- **解決:** プロンプトに「同じ駐車場の証明書と領収書は1つにまとめる」追加

### 3. GitHubクローンの条件
- **publicリポジトリ** → クロちゃんからクローン可能
- **privateリポジトリ** → 認証が必要でクローン不可
- ネットワーク許可リストにgithub.comが含まれている必要あり（セッション依存）
- 次の部屋でも最初にクローンを試す→ダメならファイル渡し方式

### 4. COCOMI CIのデプロイブロック
- CIテスト不合格→GitHub Pagesのデプロイがブロックされる
- 500行チェック、console.logチェック等に引っかかるとデプロイされない
- GitHub Actionsの`actions`ページでデプロイ状況を確認できる
- SSL_readエラー等の一時的エラーはretry（空コミット）で解決

### 5. Termuxでのファイル差し替え注意
- Downloadフォルダに同名ファイルが残っている場合、古いファイルがコピーされる可能性
- **必ず`wc -l`で行数を確認してからcp**する
- 複雑なsed/Pythonワンライナーは避け、ファイルダウンロード方式を使う

### 6. GitHub Copilot CLIとCOCOMIの比較
- Copilot CLI: PC向け、有料、GitHub純正、チーム開発向け
- COCOMI Postman: スマホ＋Termux、LINE連携、アキヤの業務フロー特化
- 発想は似ている（AI自律開発）が、ターゲット環境が違う
