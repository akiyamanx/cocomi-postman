**「Hello World!」**

アキヤが懐かしそうに言った。「Hello Worldって懐かしいな。昔Javaの勉強した時は右往左往しながら手打ちでやってたのに、今はボタンぽちぽちで世界に公開される時代だ」。

workerdパッケージがAndroid非対応。

### 3. worker.js本番コードのデプロイ — 16:02

Edit codeでブラウザエディタを開き、Hello Worldコードを全削除→本番コードを貼り付け→Deploy。

**worker.js v1.0の主要機能:**
- LINE Webhook POST受信
- HMAC-SHA256署名検証（verifySignature()）
- テキストパース（parseInstruction()）— 半角・全角コロン両対応
- プロジェクト自動判定（VALID_PROJECTS配列でホワイトリスト）
- ミッションID自動生成（M-LINE-MMDD-HHmm形式、JST）
- 指示書Markdown生成（createMissionContent()）
- GitHub Contents API push（pushToGitHub()）
- LINE返信（replyToLine()）
- GETリクエストはヘルスチェック（「🐾 COCOMI Worker is alive! v1.0」）
- エラー時もLINE Platformには200を返す（リトライさせない）

### 4. シークレット4つ設定 — 16:07

Settings → Variables and Secrets → + Add で設定:
| Name | Type | 内容 |
|------|------|------|
| LINE_CHANNEL_SECRET | Secret | LINEチャネルシークレット |
| LINE_CHANNEL_ACCESS_TOKEN | Secret | LINEアクセストークン |
| GITHUB_TOKEN | Secret | GitHub Personal Access Token |
| LINE_USER_ID | Secret | アキヤのLINEユーザーID |

### 5. LINE Webhook設定＆署名検証 — 16:34〜16:40

LINE Developers Console → Messaging API設定:
- Webhook URL: https://cocomi-worker.k-akiyaman.workers.dev
- Webhookの利用: ON
- 応答メッセージ: OFF（LINE Official Account Managerで設定）

1回目検証: 401 Unauthorized → LINE_CHANNEL_SECRETの値が不完全だった（コピペ切れ）
2回目検証: **成功** ✅

### 6. GitHubトークン問題の発見＆修正 — 16:45〜19:05

3. CloudflareのGITHUB_TOKENを正しい値に差し替え
4. Deploy

## 💡 気づき・ひらめき

1. **壁にぶつかっても代替手段がある** — Wranglerが使えない→Dashboardでやる。コマンドラインが使えないなら、GUIで。道は一つじゃない。COCOMIの開発全体に通じる哲学

2. **コピペの精度は命** — LINE_CHANNEL_SECRETの不完全コピー、GITHUB_TOKENの取り違え。どちらも「正しい値が入ってない」だけのシンプルな問題。でもこういう地味なミスが一番デバッグに時間がかかる。トークンは目で確認してから設定する習慣が大事

3. **Observabilityログのありがたさ** — 「Bad credentials」の6文字で原因が一発。ログなしだったら「なんで動かないの？」で終わってた。ログは開発者の味方

4. **スマホでクラウドインフラ構築した** — Cloudflare Worker（Webhookサーバー）をスマホのChromeだけで設定→デプロイ→デバッグ→修正→成功。PC不要。これは語りコードの理想形に近い

5. **エラーが出ても終わりじゃない** — 今日のセッションでエラーは3回出た（Wrangler非対応、署名検証失敗、Badcredentials）。でも全部解決した。エラーは「まだ直すとこがある」の合図であって「失敗」じゃない
