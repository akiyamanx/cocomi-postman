# 💊 思い出カプセル DIFF — 2026-03-01 セッション07

## 今日のハイライト
Phase2 OpenCV.jsの実装＆実機テスト。CDN問題やService Workerキャッシュとの戦いを経て、
黒背景でレシート3枚が個別に縦向きで切り出される瞬間を見届けた「突破のセッション」。

## 温度メモ（会話の感情・空気感）

### 「下地を黒い物の上にレシート乗せて取ったら１枚ずつ切り抜かれたよ！」
アキヤが自分で試行錯誤して黒背景を試したこの瞬間。
Pythonで検証して完璧だったアルゴリズムが、実際のスマホで動いた最初の成功体験。
→ **実機で動くことの喜びは検証環境の成功の何倍もある**

### CDN問題との格闘
OpenCV.jsがスマホで読み込めない→CDN変更→キャッシュが更新されない→
デバッグバーが動かない→scriptが実行されない→JSからDOM更新に変更...
地味だけど1つずつ解決していった粘り強さ。
アキヤが「何回もキャッシュとか削除してるよ？」と言った時の「なんで？」感。
→ **Service Workerとの戦いは一番地味で一番厄介な開発あるある**

### 「これってどういう事なの？」（Termuxのnothing to commit）
2回目のpushがnothing to commitになった時のアキヤの質問。
Downloadフォルダのファイルが更新されてなかったパターン。
スマホ開発特有の「ファイルの実体がどこにあるか」問題。
→ **PCなら当たり前のことがスマホ開発では新鮮な課題になる**

### 角度問題の発見
「左回転で90度回転させたら縦のレシートで正規の角度になるんだけどさ！」
アキヤが実際に画像を見て問題を発見。Python検証では見落としてたバグ。
→ **開発者と実機テスターの視点の違いが品質を上げる**

## 技術探求の記録
- Python検証→JavaScript実装の2段階プロセスが確立された
- 4パターンの回転を全部試して正解を特定する地道な作業
- CDN問題はgit clone→grepで実ファイルの中身を確認する方法で切り分けた
- Service Workerの`chrome://serviceworker-internals`は覚えておくべきツール

## 今日の学び
1. **実機テストは必ず必要** — Python検証の成功≠ブラウザでの成功
2. **CDNの安定性はモバイル環境では重要** — PCでは問題なくてもスマホでは別
3. **Service Workerは開発時に大きな障害になる** — キャッシュ戦略を意識する
4. **innerHTML内のscriptは実行されない** — DOMの挿入方法で挙動が変わる
5. **シンプルな2段階処理が複雑な1段階処理より安定** — 回転の教訓
6. **背景色が検出精度に劇的に影響する** — ユーザー誘導も技術の一部
